Google Summer of Code 2014

Welcome students! Below you'll find some information about how to apply for GSoC with MOTECH. We look forward to getting to know you.

Some Background on MOTECH

MOTECH is a platform for building mobile health applications that aim to improve health outcomes by providing better access to information. It has been used to educate pregnant women in Ghana and India about how to have healthy pregnancies, to train health workers in India, and to improve treatment regimen adherence for HIV/AIDS patients in India and Ethiopia. We have some big plans for how to expand the platform - and make it easier for non-technical users to get started with it - but our team is small and we'd love to get more contributors involved to help make those ideas into reality. You could be an important part of that!

MOTECH is written in Java and makes use of a number of open source libraries and tools. Some of the major components it makes use of are ActiveMQ, Postgres, and the Quartz Scheduler. The system is composed of a base platform that encompasses a simple API, and pluggable modules. We use OSGi for module management/lifecycle. Our front-end is built with AngularJS. A successful candidate for contribution to MOTECH will have good fluency in Java/Spring and at least some familiarity with OSGi, Postgres, Quartz, ActiveMQ, and/or Angular.

Ideas List

This ideas list is just a start - most of these are rather open-ended and will require refinement. Feel free to contact the developers to get more information about a particular idea as you are forming your proposal around it. We welcome your original ideas as well (see the following section for the student submission template). You may also view this idea list as a query in our issue tracker here. The "Prerequisites" for each idea suggests a list of technologies or concepts that a developer would ideally have experience with before choosing to work on this idea - but "learning on the job" is also expected and encouraged, so please don't feel discouraged from choosing a particular idea if you'll need to learn some new things before you can work on it.

  • MOTECH on Google App Engine

    Project Description: It isn't easy for non-technical users to get started with MOTECH - there are a lot of prerequisites to install before you can get going, and many potential MOTECH users aren't familiar with Linux. To enable easier use by non-technical users, as well as provide a hosted offering we should enable the installation and execution of MOTECH on Google App Engine.

    Prerequisites: Java/Spring/OSGi, some experience with Google App Engine or other PaaS

  • MOTECH on Arduino/Raspberry Pi

    Project Description: This is a relatively open-ended idea that requires definition, exploration, and a spec before anything can be implemented. Whomever picks up this item should explore whether we can run MOTECH on a low powered device like an Arduino or Raspberry Pi - and then propose and implement any required changes to MOTECH Platform to make this a viable self-hosting option for MOTECH customers. A complete solution would also include written documentation in the form of a how-to guide on getting up and running with MOTECH on Raspberry Pi.

    Prerequisites: Java

  • MOTECH Modules in Other Languages

    Project Description: To enable easier development of MOTECH modules, and allow reuse from organizations that have developed similar tools in other languages, we should enable MOTECH modules to be authored in other languages besides Java. e.g. Python and Ruby. A complete solution would include a Python or Ruby module sample and documentation.

    Prerequisites: Java, OSGi, techniques and tools for Java/Ruby integration and/or Java/Python integration

  • Lightweight Analytics for Non-Technical Users

    Project Description: Reporting and dashboards are frequently requested MOTECH features. While full-fledged versions of these are still a ways out on our development roadmap, we could develop some lightweight tools to bridge the gap. This project idea is to develop a lightweight set of analytical tools in advance of rolling out a more full-fledged reporting solution. The idea is that an end user can build simple queries within the MOTECH Data Services ("Seuss") UI and then take the dataset and display it in a graph. Users will be able to view data in a graph or table - this will not include the ability to define a dashboard containing multiple metrics or save the analysis and share within MOTECH.

    Prerequisites: Java, AngularJS, HTML/Javascript

  • MOTECH Dashboard

    Project Description: Today, when an admin user logs into MOTECH, the UI he sees is rather sparse. Ultimately we want a user to be presented with a helpful dashboard that reports on key indicators he cares about for his MOTECH installation. To get there, we'll start by building a generic dashboard view (with efficient data refresh and user permissions). Then we can build out a set of predefined widgets that it can display and a set of data feeds that can be connected to those widgets. Finally allow the lightweight analytics from MOTECH-586 to be set as a widget or data source.

    Prerequisites: Java, AngularJS, HTML/Javascript

  • MOTECH-CommCare Integration 3.0

    Project Description: CommCare is a system for managing health care workers and enabling them to do data collection with patients - it is part of the "MOTECH Suite" and is often deployed along with MOTECH. To enable integration of MOTECH and CommCare, we have developed a MOTECH CommCare module that provides the ability to connect to CommCare and use data from CommCare forms within MOTECH. Notably, CommCare forms can be integrated with the Tasks module. However, there are still a handful of improvements needed to make this integration seamless. This project will track (at least): surfacing CC fields in the Tasks UI; and supporting multiple versions of CC forms. The developer working on this will likely need to interact with the CommCare team at Dimagi as well.

    Prerequisites: Java, REST, CommCare

Idea Template

Have your own idea for how you'd like to contribute to MOTECH? Fantastic. Please use the following template to tell us more about it. We encourage you to engage with the community on our mailing list as you are drafting up your proposal.

Project Description: A brief description of what you plan to deliver. What does it do and who would be likely to use it? If the proposed project will include a UI, a quick and dirty UI mockup might help illustrate your idea better.

Prerequisites: What technologies or concepts should a developer be versed in to be able to effectively implement this idea?
