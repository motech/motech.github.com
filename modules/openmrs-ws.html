MoTeCH OpenMRS REST Web Services Module

Description

The MoTeCH OpenMRS REST Web Services module is an implementation of the MoTeCH MRS API that communicates with the OpenMRS application through web services.

Information for implementation

Configuration

To use the MoTeCH OpenMRS REST Web Service module, you will need the following:

  • OpenMRS 1.8.3+
  • OpenMRS REST Web Service Module 1.0+

Start by installing the OpenMRS application, and the OpenMRS REST Web Service module:

  • Installing OpenMRS
  • Installing OpenMRS Modules

Next, configure the OpenMRS REST Web Service module:

  • Login to the OpenMRS Application as an administrative user
  • Select Administration
  • Select Settings under the REST Web Services subsection
  • Configure the Rest Web Services URI Prefix
  • Optionally set the Allowed IP Addresses, Absolute Max Result Size, and Default Result Size
  • Save the settings

Next, you will need to add a Patient Identifier:

  • Login to the OpenMRS Application as an administrative user
  • Select Administration
  • Select Manage Patient Identifier Types under the Patient subsection
  • Select Add Patient Identifier Type
  • Provide a Name for the patient identifier (this name will be referred to later in the configuration of the MoTeCH module)

Finally, you need to configure the MoTeCH REST Web Service module to use your OpenMRS instance:

  • Setup and Configure the MoTeCH Project
  • Edit the openmrs.properties file under: motech-openmrs-ws/src/main/resources
  • Set openmrs.url to the URL of your OpenMRS instance
  • Set openmrs.user to a user with privileges to access the web services
  • Set openmrs.password to the password for the user
  • Set openmrs.motechIdName to the Patient Identifier Name you created in the previous step
  • Save the file, and build the MoTeCH Project

Events consumed and emitted by the OpenMRS module

Consumed events:

No events consumed by OpenMRS API module

Emitted events:

No events emitted by OpenMRS API module
