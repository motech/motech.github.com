IVR Platform Module

Description

The IVR module acts as a basic framework for implementing IVR functionality in the platform. The module provides one implementation of an IVR service that connects to an Asterick server using VoiceGlue VXML browser.

Information for implementation

The IVR module provides a common interface that IVR modules should implement, IVRService. IVRService has one method, initiateCall, which accepts a CallRequest object. CallRequests represent the information regarding a call and events associated with the results of that call.

A CallRequest has:

  • phone (String) - The phone number to call
  • timeOut (int) - The time out value for the call
  • callBackUrl (String) - The URL for VXML content
  • payload (Map) - A map for adding additional data to the call, this is where your unique identifier would be placed.
  • onSuccessEvent (MotechEvent) - The event fired when the call is successful.
  • onBusyEvent (MotechEvent) - The event fired when the call is busy.
  • onNoAnswerEvent (MotechEvent) - The event fired when the call is not answered.
  • onFailureEvent (MotechEvent) - The event fired when the call failed for some reason.

Asterisk implementations:

IVRServiceAsteriskImpl is the bean to wire IVRService with if you want to use the Asterisk server.

MotechAsteriskCallBackImpl provides CallDetailRecord keeping and events for the various stages and results of calls.
