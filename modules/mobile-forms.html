Mobile Forms Module

Description

Mobile forms module helps offline data collection and uploading to server

Information for implementation

Mobile forms api module is server component to enable form serving and data upload. Mobile forms j2me app downloads form definitions from the mobile forms api module and enables data collection. Forms are configurable thought xml and classified by form groups.

Configuration

The following properties can be changed in the mobileForms.properties file:

  • forms.config.file Points to master forms configuration.
  • forms.user.accounts: Forms server login account configuration.
  • forms.xml.form.name: Form name in xml form definition.

Events consumed and emitted by the Mobile forms api

formBeanGroup - publishes on valid form upload, client can implement listener to handle forms.

Emitted events:

{% highlight java %} Subject: "formBeanGroup" Payload: "handle.valid.xforms.group": org.motechproject.mobileforms.api.domain.FormBeanGroup {% endhighlight %}

Mobile forms extention points

  • org.motechproject.mobileforms.api.callbacks.FormPublishHandler#handleFormEvent Implement this to process forms.
