Installation

This installation guide was based on an ubuntu lucid 64bit machine set up using vagrant. It assumes you are running on the same. MOTECH will run on other operating systems but the commands required and external pckages will change. The changes shouldn't be drastic but they aren't documented here.

To get up and running with MOTECH you'll need to first install dependent software and systems, then check out and build the code and lastly deploy the artifacts. This guide will walk you through all the required steps to go from clean machine to a functioning MOTECH installation.

Dependent Software

To develop MOTECH you'll need to install some third-party software. These packages are available in most distrubtions package managers or are easily found online for download and installation.

  • Git
  • Java
  • Maven 2
  • Curl

To install these on a debian based system do the following:

{% highlight bash %} # Install required software # git-core provides git which is the source control software used in Motech # python-software-properties provides add-apt-repository # maven2 is the build system used within Motech # curl is used by some of our unit/integration tests to set up CouchDB in known states. sudo apt-get install git-core python-software-properties maven2 curl # Add the partner repository and install sun java from it sudo add-apt-repository "deb http://archive.canonical.com/ lucid partner" sudo apt-get update sudo apt-get install sun-java6-jdk # Remove Open JDK to prevent conflicts sudo apt-get remove openjdk-6-jdk openjdk-6-jre openjdk-6-jre-headless openjdk-6-jre-lib {% endhighlight %}

Dependent Systems

MOTECH makes use of a number of third-party open source systems and they each need to be installed for MOTECH to function correclty. We use Tomcat as our application container, ActiveMQ as our message queue, Quartz and MySQL for timers and CouchDB as our persitence store. Each of these packages can be downloaded from their respective sites:

  • Tomcat7
  • ActiveMQ
  • CouchDB
  • MySQL

If you are running on a 64 bit linux system you can use the following script to download and install tomcat7, ActiveMQ and CouchDB for you. The script places the systems in /opt and creates simple upstart scripts to start/stop ActiveMQ and CouchDB. It should be run as root. Trust us, it's ok. :)

MySQL should be available and easily installable via your systems package manager. MySQL 5.5 is not currently available in most package managers, but instructions for installing it are easily found online.

Once the systems are installed you can start them like this:

{% highlight bash %} sudo start activemq sudo start couchdb sudo /opt/tomcat7/bin/startup.sh {% endhighlight %}

ActiveMQ and CouchDB will log to /var/log/motech, tomcat will write it's logs to /opt/tomcat7/logs

To stop the services run these commands:

{% highlight bash %} sudo stop activemq sudo stop couchdb sudo /opt/tomcat7/bin/shutdown.sh {% endhighlight %}

Starting MySQl will vary based on the method used to install it. Typically for ubuntu it is started like this sudo /etc/init.d/mysql.server start

Code

The code for MOTECH is open souce and can be found on Google Code. You'll need a working understand of git to check it out and maven to compile it. A more detailed description of the code architecture can be found in the Overview while a description of the source tree can be found in the Source Overview of this guide.

To simply check out and compile the code, make sure all dependent software is installed then enter these commands.

{% highlight bash %} git clone https://code.google.com/p/motech/ cd motech mvn install {% endhighlight %}
